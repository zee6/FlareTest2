CobrowseDomain Permission

The "cobrowseDomain" permission grants the ability to co-browse on a certain domain (schema + fully qualified domain + optional port, i.e. http://www.unblu.com) to an agent. Whenever an agent wants to start co-browsing they need the permission to co-browse on the domain the visitor is currently viewing.

Implicit Policy

The implicit (default) policy grants the "cobrowseDomain" for all domains to any user assigned with the REGISTERED_USER role.

External Policy

In order to use the "cobrowseDomain" permission in an external policy, the implicit policy needs to be disabled with the following configuration property (see Unblu Server Configuration Reference) 

# add the setting below to your unblu configuration
com.unblu.authz.permission.cobrowseDomain.implied=false

In the external policy the "cobrowseDomain" permission has the following format:

# extract of an permissionOverlay file
  <role name="ROLENAME">
    <permissions>
      <coBrowseDomain origin="http://www.unblu.com"/>
    </permissions>
  </role>

See Advanced Authorization for details about how external policies are enabled.
