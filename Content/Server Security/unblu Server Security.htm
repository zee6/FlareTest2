unblu Server Security

This section covers on premises integrations and does not necessarily cover our cloud offering.

System Entry Path Concept

Unblu, by default, uses a set of entry paths (http request path prefix) as a high-level distinction for different kinds of requests. In typical on-premises installations these path prefixes are used as a high-level segregation for requests coming from different origins (public internet, intranet or from administrators / other systems). Based on these path prefixes, unblu grants different privileges while executing requests.

In typical installations unblu is operated behind a reverse proxy that guarantees requests are restricted according to path prefix and origin of the request.

Public (untrusted) "/unblu" Entry Path

The "/unblu" path prefix is the part of the unblu server that visitors need to have access to. Typically "/unblu" is mapped in a reverse proxy in a way that it can be accessed from within the application that is going to be co-browsed.

Restricted (trusted) "/co-unblu" Entry Path

The "/co-unblu/" path prefix is the part of the unblu server that MUST only be accessible to authenticated users (agents).

System "/sys-unblu" Entry Path

The "/sys-unblu/" path prefix is the part of the unblu server that MUST only be accessible for other systems (i.e. the unblu filter) or administrators.

Security Considerations

The "/sys-unblu/" path prefix needs to be protected in such a way that ensures that only administrators or other systems have access.
