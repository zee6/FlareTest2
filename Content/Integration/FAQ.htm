FAQ1

What can I do if the browser does not respond or responds slowly?

You can do anything you would normally do: Click the back button, reload the page, wait, even temporarily go to another page. As long as you do not close the browser (that is, terminate the browser program), co-browsing will continue.

 

FAQ2

I need a login, where can I get one?

ask the product owner of your company, if you are working for a company that has an existing unblu license. Otherwise, you can register at: https://www.unblu.com/en/products - please note that the email address will be your username - do not use info@yourcompany.com for example.

 

FAQ3

Can I add the unblu snippet dynamically ?

The code snippet is designed so that it only works when it is statically bound to a website. The snippet requires events such as "document ready" or "load". If the script does not get the events or the events have already been fired before the script is "injected", unblu is not going to work.

 

FAQ4

Why does starter.js return 301?

The "301 Moved Permanently" code is normal. The redirected script is then stored in the browser cache and only downloaded again from the server when an update on the cloud has been performed.

 

FAQ5

How can I control the buttons (unblu activation element)?

State-dependent buttons are currently only supported by custom UI implementations based on the unblu API. With the API you can define what buttons you would like to display, and consequently define the states. Details can be found in the documentation of the Developer API.

 

FAQ6

Is it possible to set the position of the control UI by custom coordinates?

Yes. You will find this and more in the Cloud Admin Application.

 

FAQ7

Is IE8 supported by unblu?

No, Internet Explorer support starts at version 10.

 

FAQ8

Why can I, as a co-visitor, not see a pdf document in my browser?

PDF files cannot be viewed in the unblu Enterprise cloud version of unblu but it works for on premises installations of unblu. However, the pdf documents cannot be co-browsed but are accessible for the co-visitor as well (as long as a pdf viewer is installed).

Enterprise Line

FAQ9

Timeout session is short. It would be nice to be able to get it parameterized.

unblu has multiple timeouts:

a) One timeout defines the duration that a PIN can be used

b) Another timeout defines how long a co-browsing session is active when there is no action by any user

Contact support@unblu.com for details.

 

FAQ10

The window asking for a PIN should also contain a field for the customer to enter a phone number that can be used to call him back.

If you would like to offer your customers a callback service you can implement your own user interface and control unblu by using the unblu API.

 

FAQ11

Can activity be logged to the statistics on Google Analytics?

Yes, Google Analytics and other customer specific logging can be implemented by using the unblu API.

 

FAQ12

Is it possible to customize the PIN dialog?

It is not possible to customize the default unblu dialogs. However, there is an option to build a custom user interface based on the unblu API according to the design of your choice.

 

FAQ13

Why does my browser indicate a secure/insecure warning?

This should not be the case. Please contact support@unblu.com to clarify.

Please take into consideration that, by default, images, CSS and other resources are transferred by the unblu server from the origin server to the co-visitor. For this reason the unblu server must have access to these resources. This is needed because our desk (co-visitor) is accessed through https, thus mixing https and http causes the co-visitor browser to indicate a secure/insecure warning.

Enterprise Line

FAQ14

How can I see all content of the page on the co-visitor browser when there is a secure/insecure warning?

This should not be the case. Please contact support@unblu.com to clarify. It is recommended to avoid mixed content. But if mixed content is used, the following applies:

IE8: On IE8 you get a popup warning asking you if only the secure resources of the page should be loaded. You can just click “NO”.

IE9: At the bottom of your browser you get a warning asking you whether to show all the content of the page. You should click on “Show All Content”.

Chrome: In the address bar you will see a shield icon on which you can click and then click on “Load anyway”.

FAQ15

FAQ16

What languages are supported by unblu?

There are many languages supported out of the box, e.g., English, German, Italian, French. If you want to support a languages that is not shipping out of the box yet, you can provide us the texts available so that we can configure the language for you.

 

FAQ17

Are IE9 / IE8 supported?

Internet Explorer version 8 is supported up to unblu version 3.4. IE9 is supported above 3.4. Please make sure that neither is running in IE7 compatibility mode, otherwise unblu may not work.

 

FAQ18

Is there a request to the unblu Javascript for every page view?

No, it is not required to load the unblu Javascript with every page view. The API works "lazily" - when the unblu code loads, it checks whether the unblu API is used on the current page. If so, it initializes it, if not, nothing happens.

 

FAQ19

Is it correct that in 'View-and-Point' mode, the co-visiitor's clicks are visualized for the visitor?

Yes, the co-visitor cannot navigate or fill-in any form. However, clicks are visualized with little circles in the customer browser. In addition, the co-visitor can use the "Marker" to highlight available elements (input, pictures, links) for the customer.

 

FAQ20

A message is displayed after the co-browsing session is terminated on both sides. Is it possible to customize this message (in appearance and text) to match customer requirements?

The message is displayed only on the client side. The texts are easily configurable by properties. The appearance can only be customized by using the unblu API.

 

FAQ21

Is it possible to have a confirmation dialog ("Do you really want to quit") before the co-browsing session is terminated?

Yes, for all business line users and for visitors using the enterprise line, this is the standard.

For co-visitors, this functionality is not yet available for the enterprise line, but it is on the roadmap.

 

FAQ22

Is placement, text and design (color) of the tab possible?

Yes, there are some parameters to position the tab. To customize the look beyond the basic configuration, the API can be used.

 

FAQ23

What happens if someone just tries out codes without being invited to a co-browsing session?

It is very unlikely to guess the PIN code by trial and error. In addition, the codes are only valid for a limited time and the number of digits and types of characters can be adjusted.

 

FAQ24

Is it possible to reuse a code?

No. The PIN can be used only once.

 

FAQ25

The PIN code is only valid for a certain time. Is it possible to invalidate it, or to configure the timeout?

Yes, it is configurable for the on-premises version of the enterprise line. For the business line, it is preset.

 

FAQ26

See On-Premises requirements

 

FAQ27

How does unblu scale?

Unblu can scale for about 2500 agents on a single multi-core server. If you are interested in setting up an unblu cluster for more than 2500 users or have any questions about scaling, please do not hesitate to contact support@unblu.com.

 

FAQ28

Is it possible to deactivate the tab?

Yes, the tab can be deactivated. Integrating unblu in your Website describes the details.

 

FAQ29

Can I use unblu in clustered environments?

Yes. There are different recommended setups for using unblu in a clustered environment. Please contact support@unblu.com for details.
