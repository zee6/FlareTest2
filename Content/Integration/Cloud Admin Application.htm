Cloud Admin Application

For all cloud accounts, the email used to register your account will provide you with access to the unblu Admin Application. Before you use the system for the first time you should configure your account in order to register your web properties with unblu, receive your account specific unblu snippet and to set your system preferences.

If you are setup as an administrator you have administrative rights to your account. Once you login you have the option to navigate to "Administer Account" under the myunblu navigation. Note that regular users do not have access to the administrative interface but directly start the co-browsing application once logged in via the unblu homepage. Consequently, you do not have to worry about regular users changing any of the configuration parameters.

Configuring your Account

We built the administrative interface in a way that all your configuration options are explained on the relevant screens, which makes it straightforward to set your parameters once you are on the screens. To give you a high-level overview, here are the navigation elements available:

Account and License

In this section you can see the product type, which you purchased, the time for which the subscription is valid and the payment state. If you have already reached the activation date and your subscription has been paid, the status of your subscription will be set to active.

User Management

In this section you can manage your users. You can add as many users as your subscription permits. You can also assign user rights and with that you can can setup additional users that have administrative rights to your account.

Account Settings

unblu allows you to determine a number of settings applied to all domains within your account. This saves you having to manage all of these settings individually on a domain level. The account settings dynamically adjust to the licensed product. For instance, you can enable in-context migration for a teams account. However, this setting will not appear in a starter account. It is important that you define the languages that you would like to activate, as the domain settings will rely on those definitions to be in place. Please read the explanations describing the account settings carefully to ensure your account works as desired.

Domain Settings

With your subscription you get the ability to manage either one or more domains within one account. In the domain settings section, you can initially define the domains for which you would like to use unblu. You can then manage the domain level settings via the 'edit' function. It is important to register all domains where you would like to use unblu in the domain section. Please note that http and https domain extensions need to be registered separately and that you should only register the root domain (i.e. http://www.unblu.com) and not specific subpages (like http://www.unblu.com/en/pricing).

Within the domain administration you can also set the text of the start session dialog. Note that the text edit option will be available for the languages that you activated in the account settings. If you do not enter your own custom text in the 'start session dialog' area, the system will use the default text. You need to avoid this as it will state that this is only a demo, so make sure you provide your own text settings.

Integration

In order to integrate the unblu functionality into your webpage, you need to embed some java script code in every page where you would like to have unblu available. The best way to achieve this is to implement the java script code on a template used throughout the site. In the Integration section of the administrative interface you will find your account-specific java script code snippet. You can use this snippet on all domains registered in your account. As an alternative to the java script code snippet, you can also use a graphical element with the unblu java script link which will call unblu. In the integration section we have provided you with a number of graphic icons which you can use as is. You can also generate your own icon by uploading a graphics file of your choice. Please note that the graphical element approach is not recommended if you would like to use chat, as you will not be able to continue a chat conversation on pages which do not have the unblu java script snippet embedded.

Analytics:

In the Statistics (Agent Desk) section you can see how often you used the co-browsing system and which agent was using it.
