Settings

Settings are basically ‘switches’. Many are actual binary switches (on, off) and can be enabled/disabled by checking a box. Some are fields that either contain, or have the ability to contain IP addresses, text (e.g., en for English or de for German), numbers or lists.

Settings > … > Settings

There are three places where those settings can be accessed and edited:

Settings > User profile > Settings Allows editing  for individual users

Settings > Account > Settings Allows editing at the Account level

Settings > Global Allows editing at the Global level

  • Settings are hierarchical. If you are not a Super Admin you will only see a subset of the available settings that are relevant to your role.
  • Registered users only see, and only have edit privileges on a very small number of self-explanatory configuration items. This number increases as one moves up the hierarchy.
  • Basic mode: Note first that there is nothing in the interface that says ‘basic’. This is because ‘basic’ is the default. What you do see is an ‘Advanced’ button.
  • In Basic mode (that is; you have not clicked the Advanced button and are therefore in ‘basic’ mode) simply hover over an item to see a description of that configuration value’s function.

Here is a picture of a typical registered user’s settings in the User Profile. Note that there are only two configuration items displaying. All other values have been set by a role higher up in the hierarchy (and those values propagate downwards, invisibly). Therefore, configuration is kept almost entirely out of the Registered user’s scope.

Hover the cursor over an item to see a description.

 Settings in Advanced Mode

For a Registered user or Supervisor there is probably no need to use the Advanced button at all. But when we move up the hierarchy through Admin to Super Admin the need for an Advanced mode becomes clearer.

Here are the same two pictures when seen from the perspective of an Admin user in Advanced mode. Select Settings > User profile > Settings then click the Advanced button.

Note that the Advanced mode provides a lot more information. In truth, even an Admin user would probably only need the basic description but Super Admins will need to access this more granular information.

Below are some more examples that are a little easier to understand. The picture below lists all the languages that have been selected in the boxes and tells us that these settings are inherited from the default settings.

Another example (below) shows what an Admin user sees when they use the Advanced mode to examine whether their configuration allows navigation on Secure Sockets Layer (SSL) sites.

The graphic was cut from Settings > User profile > Settings. Now click the Advanced button and scroll down (only Admin users and above will see this item) to Universal Co-Browsing Settings then hover over the item Allow navigation to SSL sites.

This Advanced information tells us, among other things, that the default value is false (switched off) but it has been changed at the Account level to true (switched on).

Here is the same picture taken from within the same user (Admin) account with the same role but viewing it from the ‘Account’ level. Settings > Account > Settings.

The picture, taken from higher up the hierarchy (Settings > Account), no longer tells us where the configuration was changed. This means that we can sometimes get a little more information by looking at the lower levels of the hierarchy where the (upstream) ‘source’ of changes may be viewable.

Note: To get back into ‘basic’ mode click the Advanced button again. There is no text to indicate which mode you are currently in but the Advanced mode always has filters along the top of the page.
