Migrating to a New Version of unblu

This section covers materials exclusive to the Super Admin and Admin roles.

Note: This Migration section is only relevant for on-premises installations. Cloud migrations are performed automatically, without the need for any customer action.

Note: Although the credentials for the Admin role allow them to see the Migration page we would strongly advise that any action taken here is at least overseen by the Super Admin. Missteps here can have serious and/or messy consequences.

From 4.2 onward unblu offers a new way to handle version migrations. In versions preceding 4.2 one could either use configuration files (default.properties and text.properties) or use a command line interface to set configuration and text values.

Version 4.2 now offers a GUI where you can copy and paste configurations and text in order to have those configuration and text values applied to the latest version.

Note: Migration must be performed in single steps between each version. Thus, if you want to upgrade from 3.5 to 4.2 you must first migrate from 3.5 to 4.0, then 4.0 to 4.1 then 4.1 to 4.2.

Note: You must migrate both the source configurations and the text in two separate steps.

Migrating Configuration Values
  1. From the main menu select Settings > Migration.

Note: This procedure is the same for source and text settings (given that one uses  different default.properties and text.properties files).

  1. In Config select the appropriate Version Steps radiobutton.
  2. Paste the old configuration settings into the Source box.

  1. Click the Migrate button.

In the picture above the version is already at 4.2. But this allows you to see that in the Protocol box we are getting messages that there are no changes and that the configuration should be kept.

  1. Finally, when changes are required, and flagged in the Protocol box, copy the appropriate Destination values into your default.properties file.
 Migrating Text Values
  1. From the main menu select Settings > Migration.

Note: This procedure is the same for source and text settings (given that one uses  different default.properties and text.properties files).

  1. In Text select the appropriate Version Steps radiobutton.
  2. Paste the old text settings into the Source box.

  1. Click the Migrate button.

In the picture above the version is already at 4.2. But this allows you to see that in the Protocol box we are getting messages that there are no changes to the first text configuration and it should be kept, and that the second change has been verified to be dropped.

  1. Finally, when changes are required, and flagged in the Protocol box, copy the appropriate Destination values into your text.properties file.
